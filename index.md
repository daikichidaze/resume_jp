### 1) デジタルソリューション戦略と新規事業立ち上げ＠タイ
- 期間 / 体制: 2019-2022、最大5名のクロスファンクショナルチーム
- 役割: 部門立ち上げリーダー（企画〜要件定義〜PM〜事業開発）
- 課題: 新規事業の目的と判断軸が関係者間で不統一で、企画から実装への意思決定が停滞していた
- 実施内容: 事業目的・価値仮説を整理して共通言語化 / 業務・オペレーションとKPIを再設計 / ドキュメント化とプロトタイプ提示で合意形成を主導
- 成果: 新規/受託プロジェクト10件で累計20M+ THBを創出し、企画から実装まで進める組織体制を確立

### 2) 豊田通商＠タイにおけるDX推進組織の立ち上げ
- 期間 / 体制: 2022-2024、IT子会社からの逆提案で立ち上げ、4人体制へ拡大
- 役割: 立ち上げリーダー（業務標準化・企画/要件定義支援）
- 課題: 各部門の要求が抽象的で、要件・優先順位・データ定義が揃わず、DX案件の判断が属人化していた
- 実施内容: 意思決定ポイントと判断軸を設計 / データ取得粒度・KPI・比較フォーマットを標準化 / DX/AI適用範囲を整理して段階導入計画を作成
- 成果: 年間約20プロジェクトを推進し、アイデア創出ワークショップ運営を通じて案件選定から実行までの共通プロセスを定着

### 3) AIエージェント導入支援事業の推進
- 期間 / 体制: 2024-現在、小規模チームで事業構想・プロダクト企画・導入支援を並行推進
- 役割: 事業開発リード（戦略立案・要件定義・プロダクト企画/開発・導入コンサル）
- 課題: AI活用の期待は高い一方で、目的・完了条件・導入判断基準が曖昧で、実運用に接続しにくかった
- 実施内容: 価値仮説を「効率化」から「スケーラビリティ拡張」へ再定義 / 人間とAIの役割分担とSpec設計を標準化 / 研修+共同実装+運用伴走の3フェーズ導入モデルを設計
- 成果: データサイエンティスト・マッチングプラットフォームで設計期間を約3か月→1か月に短縮し、AIエージェント導入を継続運用できる実装基盤を整備

### 4) スマートフォンGPS/センサーデータを活用した運転挙動評価アルゴリズムの開発
- 期間 / 体制: 2021-2023、日本/ベトナムの2拠点クロスボーダー体制（フルリモート・パートタイム）
- 役割: テクニカルリード兼プロジェクトマネージャー（技術方針、ロードマップ、スプリント、デリバリー管理）
- 課題: センサーデータ活用方針と評価基準が曖昧で、分散チームでの開発優先順位と検証観点が揃っていなかった
- 実施内容: GPS/慣性センサーデータのイベント検知アルゴリズムを設計・実装 / 新規アルゴリズム検証とCTO向け提案を継続 / 非同期ワークフローとコードレビュー運用を定着
- 成果: 日越2拠点のリモート体制で運転挙動イベント検知アルゴリズムを実装し、技術提案と開発運用の継続改善サイクルを確立

### 5) MaaS企画検証のためのシミュレーター開発
- 期間 / 体制: 2016-2019、3-4名チーム、総額40M JPY規模
- 役割: PM / データサイエンティスト / ソフトウェアエンジニア（要件定義〜設計〜実装〜分析）
- 課題: 新サービス要件が抽象的で、施策比較に必要な判断軸・データ定義・検証環境が不足していた
- 実施内容: 意思決定ポイントとデータ取得定義を設計 / 組合せ最適化とシミュレーション条件を設計・実装 / 分析レポートを意思決定フローに接続
- 成果: 40M JPY規模プロジェクトで事前検証可能なシミュレーターを構築し、サービス要件の意思決定精度を向上
